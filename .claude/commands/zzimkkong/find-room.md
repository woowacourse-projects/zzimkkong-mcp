조건에 맞는 찜꽁 회의실을 찾아주세요. $ARGUMENTS

## 흐름

1. **로그인 상태 확인**: `get_my_info` 툴로 로그인 상태를 확인합니다.
   - 로그인됨 → 다음 단계로 진행
   - "세션이 만료되었습니다" 응답 시:
     1. `get_last_login_method` 툴을 호출해 마지막 로그인 방법(email/github/google/none)을 확인합니다.
     2. `AskUserQuestion` 툴로 선택지를 제시합니다:
        - question: "세션이 만료되었습니다. 로그인 방법을 선택해주세요"
        - header: "로그인 방법"
        - options 4개: 마지막 로그인 방법에 해당하는 항목의 description 끝에 " (최근선택)"을 추가합니다.
          - label: "이메일/비밀번호", description: "이메일과 비밀번호로 로그인합니다."
          - label: "GitHub", description: "GitHub 계정으로 브라우저 로그인합니다."
          - label: "Google", description: "Google 계정으로 브라우저 로그인합니다."
          - label: "비로그인으로 계속", description: "코치 전용 공간은 표시되지 않습니다."
     3. 선택에 따라 로그인 툴 실행. "비로그인으로 계속" 선택 시 그대로 진행합니다.
   - "로그인되어 있지 않습니다" 응답이고 `get_last_login_method`가 none이면: 비로그인으로 계속 진행합니다.

2. **조건 수집**: 사용자가 제공한 조건을 파악합니다. 없으면 물어봅니다.
   - 날짜 (형식: `YYYY-MM-DD`, "오늘"/"내일" 등도 변환)
   - 시작·종료 시간 (형식: `HH:mm ~ HH:mm`)
   - 원하는 층: 11F / 12F / 13F / 상관없음
   - 최소 대여 시간 (예: 2시간 이상 필요)
   - 코치 전용 포함 여부 (로그인 상태에서만 가능)

3. **가용 공간 조회**: `check_availability` 툴로 해당 시간대 예약 가능 공간을 가져옵니다.

4. **조건 필터링**: 조회 결과에서 사용자 조건에 맞는 공간만 추립니다.
   - 층 조건이 있으면 해당 층만 표시
   - 최소 대여 시간이 있으면 `list_spaces`의 최대 예약 시간과 대조

5. **결과 표시**: 조건에 맞는 공간을 보여줍니다.
   - 코치 전용 공간(🔒)을 목록 상단에 우선 표시하고, 로그인된 코치 사용자라면 먼저 추천합니다.
   - 이후 층별로 정렬합니다.
   ```
   ✅ 조건에 맞는 공간 (N개)
   🔒 코치 전용 추천
     [ID] 공간이름  운영시간 | 최대 N시간
   ── 11F ──
     [ID] 공간이름  운영시간 | 최대 N시간
   ```

6. **예약 연계**: "예약할까요?"를 물어보고, 원하면 공간과 용도를 받아 `create_reservation`으로 바로 예약합니다.

## 주의사항
- 조건을 `$ARGUMENTS`로 바로 전달받은 경우 파싱해서 바로 조회합니다.
- 결과가 없으면 조건을 완화해볼 것을 제안합니다 (예: 층 조건 제거, 시간대 조정).
